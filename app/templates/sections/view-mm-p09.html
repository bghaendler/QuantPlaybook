Mathematical Methods in Financial Markets

Simulating Asset Randomness & Brownian Paths

From uniform random numbers to Brownian motion to GBM paths: the simulation pipeline, done correctly, with the standard pitfalls flagged.

1. The Pipeline: Uniforms → Normals → Paths

  1. Uniforms: a quality PRNG (Mersenne Twister, PCG64) — never rand(); seed explicitly for reproducibility.
  2. Normals: inverse-CDF $Z = N^{-1}(U)$ (Acklam/AS241) — preferred, preserves low-discrepancy structure; Box-Muller as the classical alternative.
  3. Brownian increments: $\Delta W_i = \sqrt{\Delta t}\, Z_i$ — independent, $\mathcal{N}(0, \Delta t)$; cumulative sum gives the path.
  4. Correlated assets: Cholesky factor $L$ of the correlation matrix, $\Delta W^{corr} = L\,\Delta W^{indep}$.

Properties to verify in any implementation: $W_0 = 0$, $\operatorname{Var}(W_t) = t$, increment independence (autocorrelation ≈ 0), and the $\sqrt{\Delta t}$ scaling — paths that look "too smooth" almost always scaled by $\Delta t$ by mistake.

2. Simulating GBM: Exact vs. Euler

Exact (always prefer for GBM): the SDE has a closed solution, so the discretization is exact at any step size:

$$S_{t+\Delta t} = S_t \exp\!\left[\left(\mu - \tfrac{1}{2}\sigma^2\right)\Delta t + \sigma\sqrt{\Delta t}\, Z\right].$$

Euler-Maruyama (general SDEs): $S_{t+\Delta t} = S_t(1 + \mu\Delta t + \sigma\sqrt{\Delta t}Z)$ — strong order 0.5, weak order 1; can go negative; needed only when no exact scheme exists (then consider Milstein, adding $\tfrac{1}{2}\sigma\sigma'(Z^2 - 1)\Delta t$, strong order 1.0). For CIR-type square-root processes use full truncation or the exact noncentral-χ² sampler — naive Euler on $\sqrt{v}$ is the classic blow-up.

The $-\tfrac{1}{2}\sigma^2$ matters: omit it and your simulated $\mathbb{E}[S_T]$ is right but the median path and all log-statistics are biased — the single most common student bug.

3. How Many Paths? Error Budgeting

Monte Carlo standard error: $\text{SE} = s/\sqrt{N}$ — quote it with every estimate. Two distinct errors:

  • Statistical (paths): halving needs 4× paths; attack with antithetics, control variates, Sobol QMC.
  • Discretization (steps): zero for exact GBM; $O(\Delta t)$ weak for Euler — refine steps only when the scheme is inexact or the payoff reads the path (barriers: use the Brownian-bridge crossing probability per interval rather than brute-force small steps).

Path-dependent payoffs sample the path at contract dates — simulate on the fixing grid exactly, not on an arbitrary fine grid resampled.

4. Minimal Correct NumPy Reference

rng = np.random.default_rng(42)
n_paths, n_steps = 100_000, 252
dt = T / n_steps
Z = rng.standard_normal((n_paths, n_steps))
log_paths = np.cumsum((mu - 0.5*sigma**2)*dt + sigma*np.sqrt(dt)*Z, axis=1)
S = S0 * np.exp(np.hstack([np.zeros((n_paths,1)), log_paths]))   # includes t=0
payoff = np.maximum(S[:, -1] - K, 0)
price  = np.exp(-r*T) * payoff.mean()
se     = np.exp(-r*T) * payoff.std(ddof=1) / np.sqrt(n_paths)

Vectorized (no Python loops), seeded, exact scheme, standard error reported — the four habits that separate production simulation from notebook simulation.

Key Takeaways

  • Pipeline: PRNG uniforms → inverse-CDF normals → $\sqrt{\Delta t}$-scaled increments → (Cholesky for correlation).
  • GBM has an exact scheme — use it; Euler/Milstein are for SDEs without one.
  • Never forget $-\frac{1}{2}\sigma^2$; never scale noise by $\Delta t$.
  • Budget statistical vs. discretization error separately; report the standard error always.
