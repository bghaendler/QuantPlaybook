Deep Learning & NLP: LSTM & Sentiment Analysis

Sequence models for financial time series and text: what LSTMs actually compute, where they help in finance, and how sentiment becomes a tradable feature.

1. Why Recurrent Networks for Finance?

Financial data is sequential and path-dependent: volatility clusters, order flow has memory, news arrives as a stream. A recurrent network maintains a hidden state $h_t$ summarizing the past:

$$h_t = f(W_x x_t + W_h h_{t-1} + b),$$

but plain RNNs suffer vanishing/exploding gradients — backpropagating through $T$ steps multiplies $T$ Jacobians, so long-range dependencies are unlearnable. LSTMs were designed to fix exactly this.

2. The LSTM Cell

Long Short-Term Memory (Hochreiter & Schmidhuber, 1997) adds a cell state $c_t$ — a conveyor belt modified only by elementwise gates:

$$\begin{aligned} f_t &= \sigma(W_f [h_{t-1}, x_t] + b_f) && \text{forget gate} \\ i_t &= \sigma(W_i [h_{t-1}, x_t] + b_i) && \text{input gate} \\ \tilde{c}_t &= \tanh(W_c [h_{t-1}, x_t] + b_c) && \text{candidate} \\ c_t &= f_t \odot c_{t-1} + i_t \odot \tilde{c}_t && \text{cell update} \\ o_t &= \sigma(W_o [h_{t-1}, x_t] + b_o), \quad h_t = o_t \odot \tanh(c_t) \end{aligned}$$

Because $c_t$ updates additively (gated identity path), gradients flow across hundreds of steps. The gates learn what to remember (vol level), what to overwrite (stale trends), and what to expose to the output. GRUs are the cheaper two-gate variant; Transformers have since displaced LSTMs for text but LSTMs remain competitive for modest-size numeric time series.

3. LSTMs on Market Data: Honest Expectations

  • Works: volatility forecasting (beats GARCH benchmarks modestly), order-book dynamics at short horizons, hedging policies (deep hedging — learning to hedge under frictions), feature extraction for downstream models.
  • Rarely works as advertised: direct return prediction. Signal-to-noise in daily returns is ~0.05; an expressive model happily memorizes noise. Symptoms: spectacular in-sample/backtest, random out-of-sample.
  • Discipline: walk-forward (purged) cross-validation, embargo periods, small networks, dropout/weight decay, ensembling over seeds, and benchmarking against ridge regression and GARCH — if the LSTM can't beat those, it is overfit decoration.

4. Sentiment Analysis: Text as a Factor

Pipeline: source (news wires, earnings calls, filings, social media) → entity tagging → sentiment scoring → aggregation into a time series → orthogonalization against known factors → signal test.

  • Lexicon era: Loughran-McDonald dictionary — finance-specific word lists (in general English, "liability" is neutral; in filings it isn't). Transparent, fast, weak.
  • Embedding/LSTM era: word vectors + recurrent encoders capture negation and context ("failed to miss estimates").
  • Transformer era: FinBERT and LLM-based scoring dominate accuracy; the modeling edge has shifted to data breadth, latency, and entity resolution rather than the classifier.
  • Stylized findings: news sentiment predicts short-horizon (days) drift with rapid decay; earnings-call tone shifts predict post-announcement drift; aggregate sentiment is largely a fast-moving risk-on/off proxy — decays fast, crowds quickly.

5. A Reference Architecture

  1. Inputs: 60-day window of standardized features (returns, realized vol, volume, sentiment score).
  2. Model: 1–2 LSTM layers (32–64 units), dropout 0.2–0.5, dense head; classification target (up/flat/down or vol quantile) is usually more robust than regression on returns.
  3. Training: rolling windows, early stopping on a chronologically later validation block; never shuffle across time.
  4. Evaluation: transaction-cost-adjusted PnL and turnover, not accuracy; compare to the linear baseline on identical features.

Key Takeaways

  • LSTM gates create an additive memory path that defeats vanishing gradients.
  • Best financial uses: volatility, microstructure, hedging policies — not naive return prediction.
  • Finance text needs finance models: Loughran-McDonald → FinBERT → LLM scoring.
  • Sentiment is a fast-decaying, crowdable alpha; engineering (entities, latency) is the moat.
  • Chronological validation with embargoes is non-negotiable — leakage is the field's default failure.
