Algorithmic Adjoint Differentiation

Elemental Operations & Computation Graphs

The DAG picture that unifies all of AD: nodes, local partials, and the two sweep directions — worked end-to-end on a Black-Scholes-flavored example.

1. Programs as DAGs

Decompose execution into elemental operations — unary/binary primitives with known partials. Each intermediate $v_i = \phi_i(v_{j \prec i})$ is a node; edges carry the local partials $\partial v_i/\partial v_j$. Example function (a stylized $d_1$ computation):

$$y = \frac{\ln(S/K) + \sigma^2/2}{\sigma}$$ 
v1 = S / K          ∂v1/∂S = 1/K        ∂v1/∂K = -S/K²
v2 = ln(v1)         ∂v2/∂v1 = 1/v1
v3 = σ * σ          ∂v3/∂σ = 2σ
v4 = v3 / 2         ∂v4/∂v3 = 1/2
v5 = v2 + v4        ∂v5/∂v2 = ∂v5/∂v4 = 1
y  = v5 / σ         ∂y/∂v5 = 1/σ        ∂y/∂σ = -v5/σ²

The graph is a DAG (σ feeds two nodes — "fan-out"); the full derivative is the sum over all paths from input to output of the product of edge partials (Baur-Strassen view).

2. Forward Sweep on the Graph

Seed $\dot{S} = 1$ (others 0); push tangents through every node in execution order: $\dot{v}_i = \sum_j \frac{\partial v_i}{\partial v_j}\dot{v}_j$.

With $S = 100, K = 95, \sigma = 0.2$: $\dot{v}_1 = 1/95$, $\dot{v}_2 = \frac{1}{v_1}\dot{v}_1 = 1/100$, $\dot{v}_5 = 0.01$, $\dot{y} = 0.01/0.2 = \mathbf{0.05} = \partial y/\partial S = \frac{1}{S\sigma}$ ✓. One sweep per input — to get $\partial y/\partial K$ and $\partial y/\partial\sigma$, re-run with the other seeds (or vector mode, Part 05).

3. Reverse Sweep on the Graph

Seed $\bar{y} = 1$; pull adjoints backwards: $\bar{v}_j \mathrel{+}= \frac{\partial v_i}{\partial v_j}\bar{v}_i$ (note the accumulation — fan-out nodes sum contributions over all uses).

ȳ  = 1
v̄5 = ȳ/σ = 5            σ̄ += ȳ·(-v5/σ²)        ← y-node
v̄2 = v̄5 = 5             v̄4 = v̄5 = 5            ← sum node
v̄3 = v̄4/2 = 2.5         σ̄ += v̄3·2σ = +1        ← σ's 2nd use
v̄1 = v̄2/v1 = 4.75
S̄  = v̄1/K = 0.05        K̄ = v̄1·(-S/K²) ≈ -0.0526

One backward sweep delivered all three sensitivities ($\bar S, \bar K, \bar\sigma$) — the $O(1)$-per-output miracle, visible at graph scale. The missed += at fan-out nodes is the single most common hand-coded-adjoint bug.

4. Graph-Level Insights

  • Memory = live nodes: reverse needs intermediate values ($v_1, \sigma, \ldots$) available backwards — store (tape) or recompute (checkpointing, Part 06): the graph makes the trade visible.
  • Mode choice = matrix shape: Jacobian of $f: \mathbb{R}^n \to \mathbb{R}^m$ costs $n$ forward or $m$ reverse sweeps; cross-country/optimal elimination orders (interleaving) are NP-hard in general — heuristics only.
  • ML framing: this DAG is TensorFlow/PyTorch's computation graph; backprop's $\delta$'s are the $\bar{v}$'s; "gradient accumulation" is the fan-out rule.
  • Custom nodes: any subgraph with a known mathematical derivative (a solve, an interpolation, a BS formula) can collapse into one super-node with hand-supplied partials — the optimization that powers Part 02's hybrid pattern.

Key Takeaways

  • Execution = DAG of elementals; derivatives = sums over paths of products of local partials.
  • Forward pushes tangents (per input); reverse pulls adjoints (per output) with += at every fan-out.
  • The reverse sweep's need for stored intermediates is AD's defining engineering constraint.
  • Collapsing subgraphs into super-nodes with analytic partials is the key production optimization.
