Notes on the QuantLib Object Model

        
    
            
  • Instrument / Engine separation: a VanillaOption holds payoff + exercise; the pricing model lives in the engine (AnalyticEuropeanEngine, BaroneAdesiWhaleyApproximationEngine, ...). Swap the engine, keep the instrument.
    • 
            
  • Term structures as handles: curves and vols are wrapped in Handles so market data updates re-price instruments automatically.
    • 
            
  • Day counting and calendars matter: the generated code uses Actual365Fixed and a flat calendar — align these with your market conventions before production use.
    • 
            
  • Install with pip install QuantLib (the SWIG Python wheel).
