1. General Optimization Template

Optimization is the mathematical process of finding the best possible value of an objective function subject to specified constraints. In finance, "best" means maximizing returns/utility or minimizing risk/cost.

$$\min_{x_1, \dots, x_n} f(x_1, \dots, x_n)$$ $$\text{subject to: } \begin{cases} g_j(x_1, \dots, x_n) \leq b_j \quad (j = 1, \dots, m_{\text{ineq}}) \\ h_k(x_1, \dots, x_n) = c_k \quad (k = 1, \dots, m_{\text{eq}}) \end{cases}$$

Three Core Pillars:

  • Objective Function ($f$): The metric we want to optimize (e.g. portfolio variance, Sharpe ratio, utility).
  • Decision Variables ($x_i$): The variables we control (the "dials" we can turn, e.g. asset weights $w_i$).
  • Constraints: Restricting boundaries (e.g. no short-selling $w_i \geq 0$, or the fully-invested budget equation $\sum w_i = 1$).

2. Three Practical Formulation Tips

💡 Tip 1: Maximization Reflection

To solve a maximization problem using standard minimization solvers:

$$\max f(x) = -\min(-f(x))$$

💡 Tip 2: Affine Transformations

Adding a constant $a$ or multiplying by a positive constant $b > 0$ does not change the optimal coordinate $x^*$:

$$\arg\max_x (a + b f(x)) = \arg\max_x f(x)$$

💡 Tip 3: Excel Solver Target Cells

Setting a cell to a target value is mathematically modeled by minimizing the norm or distance:

$$\min_x \|f(x) - \text{target}\|^2$$

1. Gradient, Hessian & Extrema Conditions

For an n-dimensional decision vector $\mathbf{x} \in \mathbb{R}^n$, unconstrained smooth functions can be classified using partial derivative structures:

  • The Gradient ($\nabla f(\mathbf{x})$): The column vector of first partial derivatives: $$\nabla f(\mathbf{x}) = \begin{pmatrix} \frac{\partial f}{\partial x_1} & \dots & \frac{\partial f}{\partial x_n} \end{pmatrix}^{\mathrm{T}}$$
  • The Hessian Matrix ($H f(\mathbf{x})$): The symmetric matrix of all second partial derivatives (the Jacobian of the gradient): $$H f(\mathbf{x}) = \begin{pmatrix} \frac{\partial^2 f}{\partial x_i \partial x_j} \end{pmatrix}_{n \times n}$$
First-Order Necessary Condition (FOC): At an optimal extrema $\mathbf{x}^*$, the gradient must vanish: $\nabla f(\mathbf{x}^*) = 0$.
Second-Order Sufficient Condition (SOC):
  • If $H f(\mathbf{x}^*)$ is strictly positive definite, $\mathbf{x}^*$ is a local minimum.
  • If $H f(\mathbf{x}^*)$ is strictly negative definite, $\mathbf{x}^*$ is a local maximum.

Equality Constraints and the Lagrangian

When our decision variables are bound by $m$ exact equality relationships (such as budget constraints or asset limits), we introduce Lagrange multipliers $\lambda_j$ to transform the constrained problem into an unconstrained system in a higher dimension:

$$L(\mathbf{x}, \boldsymbol{\lambda}) = f(\mathbf{x}) + \sum_{j=1}^{m} \lambda_j \left( g_j(\mathbf{x}) - b_j \right)$$

This allows us to identify optimal coordinates by solving the system of first partial derivatives set to zero for all $n+m$ parameters:

First-Order Condition (w.r.t. decision variables):

$$\frac{\partial L}{\partial x_i} = \frac{\partial f}{\partial x_i} + \sum_{j=1}^m \lambda_j \frac{\partial g_j}{\partial x_i} = 0$$

Constraint Recovery (w.r.t. multipliers):

$$\frac{\partial L}{\partial \lambda_j} = g_j(\mathbf{x}) - b_j = 0$$

Portfolio Optimization Case Roadmaps

Case A: Return Target, No Risk-Free Asset

Objective: $\min_w \frac{1}{2} w^{\mathrm{T}}\Sigma w$ s.t. $w^{\mathrm{T}}\mu = m, w^{\mathrm{T}}\mathbf{1} = 1$.

Defining constants $A = \mathbf{1}^{\mathrm{T}}\Sigma^{-1}\mathbf{1}, B = \mu^{\mathrm{T}}\Sigma^{-1}\mathbf{1}, C = \mu^{\mathrm{T}}\Sigma^{-1}\mu$:

$$w^* = \frac{1}{AC - B^2} \Sigma^{-1} \left[ (A\mu - B\mathbf{1})m + (C\mathbf{1} - B\mu) \right]$$

Case B: Global Minimum-Variance Portfolio (GMVP)

The unconstrained minimum-variance portfolio weight vector and minimum variance:

$$w_g = \frac{\Sigma^{-1}\mathbf{1}}{\mathbf{1}^{\mathrm{T}}\Sigma^{-1}\mathbf{1}}, \quad \sigma_g^2 = \frac{1}{A}$$

Case C: Risky & Risk-Free Asset Combinations

Optimal weight vector and Tangency Portfolio weights:

$$w^* = \frac{(m-r)\Sigma^{-1}(\mu-r\mathbf{1})}{(\mu-r\mathbf{1})^{\mathrm{T}}\Sigma^{-1}(\mu-r\mathbf{1})}$$
$$w_t = \frac{\Sigma^{-1}(\mu - r\mathbf{1})}{B - Ar}$$

Black-Litterman Model Roadmap

Black-Litterman resolves the extreme sensitivity of Markowitz weights by blending a CAPM-implied market equilibrium (prior) with subjective investor views using a Bayesian framework:

$$\hat{\mu}_I = \left[ (\tau \Sigma)^{-1} + P^{\mathrm{T}} \Omega^{-1} P \right]^{-1} \left[ (\tau \Sigma)^{-1} \Pi + P^{\mathrm{T}} \Omega^{-1} Q \right]$$
Step 1: Reverse Optimization to extract Prior ($\Pi$): $$\Pi = \lambda_{\text{mkt}} \Sigma w_{\text{mkt}}$$
Step 2: Define the Views Matrix ($P$), Return ($Q$), and Confidence ($\Omega$): $P$ maps views to specific assets; $Q$ holds view expected returns; $\Omega$ is a diagonal covariance matrix of view uncertainty.
Step 3: Asset Weights Allocation: Apply the new posterior excess return vector to standard allocation: $$w^* = \frac{1}{\lambda} \Sigma^{-1} \hat{\mu}_I$$

Active Portfolio Management & Overlay Optimization

For active managers evaluated against a benchmark index $R_B$ with weights $w_B$, we optimize the active weights $\Delta \mathbf{w} = \mathbf{w}_P - \mathbf{w}_B$ rather than absolute weights. This constitutes a zero-net-weight overlay portfolio ($\Delta \mathbf{w}^{\mathrm{T}}\mathbf{1} = 0$):

$$\max_{\Delta \mathbf{w}} \left( \Delta \mathbf{w}^{\mathrm{T}} \boldsymbol{\mu} - \frac{\lambda}{2} \Delta \mathbf{w}^{\mathrm{T}} \mathbf{\Sigma} \Delta \mathbf{w} \right) \quad \text{subject to: } \Delta \mathbf{w}^{\mathrm{T}}\mathbf{1} = 0$$
💡 **The Active Frontier**: Active management focuses entirely on maximizing the *Information Ratio (IR)*, which evaluates active return (tracking error return) divided by active risk (tracking error volatility). Because the active weights sum to zero, the risk-free asset does not appear in the active constraint set, representing a purely self-financing excess return overlay.
