Mathematical Methods in Financial Markets

Practical Stochastic Path Generation in Excel

GBM, Vasicek and CIR paths with nothing but native worksheet functions — plus the discipline (seeding, recalculation control, error reporting) that separates a demo from a tool.

1. The Building Blocks

  • Uniforms: RAND() — volatile (recalculates on every edit; F9 = new scenario). For reproducibility, paste-values a block of draws or seed a Wichmann-Hill style LCG in cells.
  • Normals: =NORM.S.INV(RAND()) — the inverse-CDF method (Part 09's preferred route). Avoid the legacy 12-uniform trick; it truncates tails at ±6.
  • Time grid: daily steps $\Delta t = 1/252$ in a column; parameters ($S_0$, μ, σ, κ, θ) in absolute-referenced cells ($B$1 style) so paths fill down cleanly.

Plain English Notes

The Big Picture: Generating random paths in Excel requires combining the uniform RAND() function with the normal CDF transformation.

  • RAND(): Generates flat, uniform random numbers between 0 and 1.
  • NORM.S.INV: Transforms uniform numbers into standard normal bell curves.

2. The Three Recipes

With parameters in B1:B5 ($S_0/r_0$, μ, σ, κ, θ), $\Delta t$ in B6, and the previous value in C1:

GBM (exact — Part 09):
C2 =C1*EXP(($B$2-$B$3^2/2)*$B$6 + $B$3*SQRT($B$6)*NORM.S.INV(RAND()))

Vasicek (exact discretization):
C2 =$B$5 + (C1-$B$5)*EXP(-$B$4*$B$6)
     + $B$3*SQRT((1-EXP(-2*$B$4*$B$6))/(2*$B$4))*NORM.S.INV(RAND())

CIR (full-truncation Euler — keeps the rate non-negative):
C2 =C1 + $B$4*($B$5-MAX(C1,0))*$B$6
     + $B$3*SQRT(MAX(C1,0))*SQRT($B$6)*NORM.S.INV(RAND())

Fill down 252 rows = one year. GBM and Vasicek use their exact transition laws (no discretization error at any step size); CIR has no elementary exact scheme, so truncated Euler with MAX(·,0) is the worksheet-grade compromise — check the Feller ratio $2\kappa\theta/\sigma^2$ in a cell and flag it red below 1.

Plain English Notes

The Big Picture: Asset paths can be generated in Excel using different discretization schemes (Exact lognormal, Euler, or Milstein).

  • Exact Recipe: Excel cells calculate paths using the exact lognormal SDE solution, which has zero step error.
  • Euler Recipe: Steps the price linearly. It is simpler but introduces errors if the time steps are too large.

3. From One Path to a Monte Carlo

  • Layout: paths across columns (C:GT for 200 paths), time down rows — one block of NORM.S.INV(RAND()) draws, one block of prices referencing it, so antithetics are just a third block using the negated draws.
  • Pricing: terminal row → =AVERAGE(MAX-payoff row) discounted; standard error =STDEV.S(payoffs)/SQRT(n) — quote it, always (Part 09's rule).
  • Recalculation control: Formulas → Calculation Options → Manual; F9 to redraw. A Data Table (What-If Analysis) with a blank input cell re-runs the sheet $n$ times — the classic trick for thousands of scenarios without VBA.
  • Convergence check: running mean =AVERAGE($H$2:H2) filled down, charted — visually confirms the $1/\sqrt{N}$ settle and catches bugs (a drifting running mean means a wrong drift term).

Plain English Notes

The Big Picture: To run a Monte Carlo simulation in Excel, we use Data Tables to copy and recalculate a single path thousands of times.

  • Data Tables: Reruns the entire sheet for different random seeds without writing VBA macros.
  • Average Payoff: The final option price is calculated by taking the average of the discounted payoffs across all table rows.

4. Excel's Limits (Know When to Leave)

  • RAND() is fine for teaching; it is not a production PRNG — period and statistical quality lag Mersenne/PCG (Part 09); regulated risk numbers should come from a coded engine.
  • Volatility of recalculation makes audit trails hard: paste-values the draws for any result you intend to keep.
  • Practical ceiling ≈ 10⁴–10⁵ cells of path data before the workbook crawls; beyond that, the same recipes are five lines of NumPy (Part 09's reference code).
  • Excel remains unbeatable for communication: a live path chart with parameter spinners teaches mean reversion faster than any equation — build intuition here, ship code elsewhere.

Plain English Notes

The Big Picture: Excel is great for quick models but fails for large simulations due to slow calculation speeds and size limits.

  • Speed Bottleneck: Running 100,000 paths with multiple assets will freeze Excel; quants switch to Python or C++.
  • Auditability: Spreadsheet formulas are easily overwritten by accident, making them risky for production financial audits.

Key Takeaways

  • NORM.S.INV(RAND()) + exact transition formulas = correct GBM/Vasicek paths in one formula each.
  • CIR needs the MAX(·,0) truncation guard; watch the Feller condition in a flagged cell.
  • Data Tables give VBA-free Monte Carlo; manual calculation + pasted draws give reproducibility.
  • Excel for intuition and prototypes; NumPy for anything measured in paths × steps.
