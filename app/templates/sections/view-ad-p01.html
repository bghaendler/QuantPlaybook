Algorithmic Adjoint Differentiation — Part 01

Source Code Transformation Principles

Derivatives of programs, not formulas: why AD beats finite differences and symbolic algebra, and the two modes every AD tool implements.

1. Three Ways to Differentiate a Pricer

MethodCost for n inputsAccuracy
Finite differences ("bumping")$(n+1)\times$ pricer cost$O(h)$/$O(h^2)$ truncation vs. cancellation — the step-size dilemma; ~half machine precision at best
Symbolic algebraExpression swellExact, but can't differentiate loops, branches, solvers — i.e. real code

The adjoint promise is the headline: a 10,000-risk-factor XVA book gets its full gradient for a constant multiple of one valuation — the technology behind real-time risk on banks' counterparty desks.

2. The Core Idea: Programs Are Compositions

Any pricer is a sequence of elemental steps $x \mapsto v_1 \mapsto v_2 \mapsto \dots \mapsto y$, each elementary ($+, \times, \exp, \ldots$) with a known local derivative. The chain rule composes them:

$$\frac{\partial y}{\partial x} = \frac{\partial y}{\partial v_k}\cdot\frac{\partial v_k}{\partial v_{k-1}}\cdots\frac{\partial v_1}{\partial x}.$$

Forward (tangent) mode multiplies this product left-to-right alongside the original execution — one sweep per input direction. Reverse (adjoint) mode multiplies right-to-left — one sweep per output, propagating "sensitivities of the result" backwards. Pricing has many inputs (market data) and one output (PV): adjoint wins. Calibration residuals (many outputs, few parameters) can favor forward — mode choice is a matrix-shape question.

3. Two Implementation Families

  • Source-code transformation (SCT): a tool parses your code and emits new code computing derivatives (Tapenade, the classic Fortran/C lineage). Fast (compiler-optimized), but requires analyzable source — covered in Part 02's comparison.
  • Operator overloading (OO): replace double with an active type that records or propagates derivatives through every operation (C++ templates: dco/c++, CppAD, ADOL-C; Python: JAX/PyTorch autograd). Easy adoption, runtime overhead, tape memory — Parts 02 and 06–08 dissect it.

Deep learning's backpropagation is reverse-mode AD specialized to neural nets — quant finance and ML run on the same engine (a point the ML masterclass exploits).

Key Takeaways

  • AD differentiates code exactly: no bump-size dilemma, no expression swell.
  • Forward mode costs per input; adjoint costs per output — full gradients in ~3–5× one valuation.
  • SCT compiles derivatives; operator overloading records them — the trade is speed vs. adoption ease.
  • Backprop = adjoint AD: every ML framework is a quant risk engine in disguise (and vice versa).
