Black-Litterman Model

A Bayesian repair of Markowitz optimization: start from market-implied equilibrium returns, then tilt them by investor views with controlled confidence.

1. The Problem with Raw Markowitz

Mean-variance optimization is an error-maximizing machine: tiny changes in expected-return inputs produce wild, concentrated, often short portfolios. Historical mean returns are estimated with enormous noise (standard error $\sigma/\sqrt{T}$ shrinks painfully slowly), so feeding them directly into the optimizer produces unusable allocations. Black & Litterman (1990, Goldman Sachs) replace the fragile input with a disciplined prior.

2. Step 1 â€” Implied Equilibrium Returns

Instead of estimating returns, reverse-optimize: ask what expected returns would make the observed market-cap weights $w_{mkt}$ optimal. With risk-aversion $\delta$ and covariance $\Sigma$:

$$\Pi = \delta\, \Sigma\, w_{mkt}.$$

$\Pi$ are the equilibrium excess returns â€” the CAPM-consistent prior. If you hold no views, the model hands back the market portfolio: a sensible, fully-invested default with no estimation error. $\delta$ is typically calibrated as $\delta = (\mathbb{E}[R_M] - r_f)/\sigma_M^2 \approx 2.5$.

3. Step 2 â€” Expressing Views

Views are linear statements about expected returns, packaged as $P \mu = Q + \varepsilon$, $\varepsilon \sim \mathcal{N}(0, \Omega)$:

  • Absolute view: "German equities will return 6%" â†’ row of $P$ has 1 on Germany, $Q = 6\%$.
  • Relative view: "Tech will outperform utilities by 2%" â†’ $+1$ on tech, $-1$ on utilities, $Q = 2\%$.

$\Omega$ (view uncertainty, usually diagonal) encodes confidence: small $\omega_k$ = strong conviction. A common default is $\Omega = \operatorname{diag}(P (\tau\Sigma) P^{\top})$, with Idzorek's method mapping intuitive 0â€“100% confidence levels onto $\Omega$.

4. Step 3 â€” The Bayesian Blend

Treat $\Pi$ as prior ($\mu \sim \mathcal{N}(\Pi, \tau\Sigma)$, with $\tau$ a small scalar â‰ˆ 0.01â€“0.05) and views as noisy observations. The posterior mean â€” the Black-Litterman expected returns â€” is the precision-weighted average:

$$\mu_{BL} = \left[(\tau\Sigma)^{-1} + P^{\top}\Omega^{-1}P\right]^{-1}\left[(\tau\Sigma)^{-1}\Pi + P^{\top}\Omega^{-1}Q\right].$$

Equivalent intuition-friendly form:

$$\mu_{BL} = \Pi + \tau\Sigma P^{\top}\left(P\tau\Sigma P^{\top} + \Omega\right)^{-1}(Q - P\Pi),$$

i.e. equilibrium plus a correction proportional to the surprise $(Q - P\Pi)$ of each view, distributed across all correlated assets via $\Sigma$. Feeding $\mu_{BL}$ (and posterior covariance) into the optimizer yields moderate, intuitive tilts away from market weights â€” only where views exist, sized by confidence.

5. Worked Mini-Example

Two assets, equilibrium $\Pi = (4\%,\ 6\%)$. View: asset 1 outperforms asset 2 by 1% ($P = [1, -1]$, $Q = 1\%$) with moderate confidence. Equilibrium implies $P\Pi = -2\%$, so the view surprise is $+3\%$. The posterior shifts asset 1 up and asset 2 down â€” but only partially (say to $5.1\%$ and $5.2\%$ for moderate confidence), and the resulting portfolio overweights asset 1 modestly instead of going 100/0 as naive Markowitz would. Key Takeaways

  • Reverse optimization gives a noise-free starting point: $\Pi = \delta \Sigma w_{mkt}$.
  • Views are linear constraints with explicit uncertainty $\Omega$; relative views are the natural use case.
  • Posterior returns are a precision-weighted blend of equilibrium and views.
  • No views â‡’ hold the market. Stronger views â‡’ larger, but still diversified, tilts.
  • BL fixes input sensitivity, not the optimizer itself â€” covariance estimation still matters.
