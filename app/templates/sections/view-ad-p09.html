2. Integration Patterns for a Quant Library

  • Scalar-template the analytics core (template<class Real>): one source tree serves double, tangent, and adjoint types — the dco/CppAD adoption route; keep I/O, logging and infrastructure passive.
  • Registered primitives at module boundaries: curve bootstrap (implicit-function adjoint of the solved system), Brent/Newton solvers, SVD/regressions, interpolators — each exposes custom_vjp-style analytic derivatives; tools differentiate only the glue between them.
  • Payoff smoothing policy: digitals/barriers carry zero a.s. pathwise derivatives but nonzero true Greeks — institutionalize smoothed payoffs (call-spread digitals, barrier shifts) or hybrid likelihood-ratio terms in the library, not per-trade ad hoc.
  • Regression-tested derivative builds: every release runs tangent-vs-bump, adjoint-vs-tangent dot products, and golden gradients on a trade zoo — generated code changes with the tool version; CI is the safety net.

3. Flagship Applications

  • XVA risk (the killer app): CVA's gradient over thousands of curve/vol nodes at ~4–10× one CVA valuation — overnight batches became intraday; Giles-Glasserman's "Smoking adjoints" (2006) started the avalanche.
  • Calibration: exact gradients/Jacobians for Levenberg-Marquardt — faster, more stable convergence than bumped Jacobians (and Part 05's vector-forward serves the few-parameter side).
  • Capital & XVA second-order: sensitivities of ES/SIMM margins; second-order needs Part 10's machinery.
  • ML-in-finance crossover: differentiable pricers embedded in neural training loops (deep hedging, model calibration networks) — possible precisely because the pricer exposes vjp's.

4. Choosing, Pragmatically

  • Existing large C++ library → OO templating (dco-style) + custom boundary adjoints; budget the Part 06 memory plan first.
  • New research code / GPU-first → JAX (or PyTorch): AD, JIT and vectorization in one tool; port hot paths later if needed.
  • Legacy Fortran/C kernels → Tapenade/Enzyme generation with hand-review of the emitted reversal.
  • Whatever the tool: the concepts are invariant — tapes, seeds, fan-out accumulation, checkpointing, custom adjoints. Tools change; Parts 03–08 don't.

Key Takeaways

  • Landscape: Tapenade/Enzyme generate; dco/CppAD tape; JAX traces and compiles — same mathematics throughout.
  • Template the core, register analytic adjoints at solver/regression boundaries, and standardize payoff smoothing.
  • XVA gradients and calibration Jacobians are the proven payoffs; deep-hedging pipelines are the growth area.
  • Pick tools by codebase reality; protect correctness with dot-product and golden-gradient CI.
